## ما هو Cursor؟

Cursor هو محرر كود مبني على VS Code لكن مع AI مدمج بعمق في كل جزء من تجربة البرمجة. بدلاً من التنقل بين المحرر و ChatGPT، كل شيء يحدث في مكان واحد.

## لماذا Cursor وليس GitHub Copilot؟

| الميزة | Cursor | Copilot |
|--------|--------|---------|
| إكمال كود | ✅ ممتاز | ✅ ممتاز |
| محادثة مع الكود | ✅ مدمج | ⚠️ محدود |
| تعديل ملفات متعددة | ✅ نعم | ❌ لا |
| فهم المشروع كاملاً | ✅ نعم | ⚠️ جزئي |
| مجاني | ⚠️ 2000 إكمال/شهر | ❌ 10$/شهر |

## كيف تبدأ

### 1. التثبيت
حمّل Cursor من الموقع الرسمي. سيستورد إعدادات VS Code تلقائياً.

### 2. الإعداد الأول
- سجّل حساب مجاني
- اختر نموذج AI (Claude أو GPT-4)
- فعّل "Auto-complete" و "Chat"

## الميزات الأساسية

### Tab — الإكمال الذكي
أقوى ميزة. ابدأ بكتابة أي شيء واضغط Tab:

```python
# اكتب تعليق يصف ما تريد
# مثال: دالة تحسب المسافة بين نقطتين
# اضغط Tab...

def calculate_distance(x1, y1, x2, y2):
    return ((x2 - x1) ** 2 + (y2 - y1) ** 2) ** 0.5
```

Cursor يفهم السياق ويكمل الكود بذكاء.

### Cmd+K — التعديل الذكي
حدد كوداً واضغط Cmd+K واكتب ما تريد تغييره:

- "حوّل هذا لـ async/await"
- "أضف error handling"
- "حسّن الأداء"
- "أضف TypeScript types"

### Cmd+L — المحادثة
افتح نافذة المحادثة واسأل أي سؤال عن كودك:

- "اشرح هذه الدالة"
- "لماذا هذا البيت يعطي خطأ؟"
- "كيف أربط هذا بقاعدة البيانات؟"
- "اكتب tests لهذا الملف"

### @-mentions
في المحادثة، يمكنك الإشارة لملفات محددة:

```
@src/api/auth.ts لماذا التوكن ينتهي بسرعة؟
@package.json ما الـ dependencies الزائدة؟
```

## سيناريوهات عملية

### بناء API كامل
```
Cmd+K: أنشئ Express API مع endpoints لـ:
- GET /articles - جلب كل المقالات
- GET /articles/:id - مقال واحد
- POST /articles - إضافة مقال
- PUT /articles/:id - تعديل
- DELETE /articles/:id - حذف
استخدم TypeScript و Zod للـ validation
```

النتيجة: API كامل جاهز في دقيقة.

### إصلاح خطأ
```
Cmd+L: أحصل على هذا الخطأ:
TypeError: Cannot read property 'map' of undefined
at ArticleList (src/components/ArticleList.tsx:15)

كيف أصلحه؟
```

Cursor يقرأ الملف المذكور ويقترح الحل.

### Refactoring
حدد ملفاً كاملاً واضغط Cmd+K:
```
حوّل هذا المكوّن من class component لـ functional component
مع hooks. حافظ على نفس الوظائف.
```

## نصائح للمحترفين

### 1. استخدم .cursorrules
أنشئ ملف `.cursorrules` في root المشروع:

```
أنت مطور TypeScript خبير.
استخدم دائماً:
- TypeScript strict mode
- Zod للـ validation
- Error handling مع custom errors
- JSDoc للـ public APIs
تجنّب:
- any type
- console.log (استخدم logger)
- تعليقات بديهية
```

هذا يضبط سلوك AI في كل المحادثات.

### 2. كن محدداً
```
❌ "اكتب دالة تسجيل دخول"
✅ "اكتب دالة تسجيل دخول بـ email/password.
   - تستخدم bcrypt للتحقق
   - ترجع JWT token (صلاحية 7 أيام)
   - تسجّل محاولات الدخول الفاشلة
   - تقفل الحساب بعد 5 محاولات"
```

### 3. راجع دائماً
AI ليس معصوماً. راجع الكود المُولّد:
- هل يتبع أنماط المشروع؟
- هل يعالج حالات الحافة (edge cases)؟
- هل يحتوي ثغرات أمنية؟

## الخلاصة

Cursor يغيّر قواعد اللعبة في البرمجة. المبرمج الذي يستخدمه بمهارة ينجز في ساعة ما كان يأخذ يوماً كاملاً. لكن تذكّر: AI يكتب الكود، أنت تصمم الحلول. المهارة الحقيقية ليست في كتابة الكود بل في معرفة ما تبني ولماذا.

**نصيحة عَقول:** ابدأ بمشروع جانبي صغير مع Cursor. جرّب كل الميزات وتعرّف على نقاط قوته. خلال أسبوع ستتعجب كيف كنت تبرمج بدونه.
